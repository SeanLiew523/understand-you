# Style A/B Test

## Goal
Calibrate by examples, not vague adjectives.

## Rule
Give 2 short versions only.
Both should be concise.
Ask which feels closer.

## Template
我不给你抽象问“你喜欢什么风格”，我直接给你两版，你选更接近你的就行。

### 版本 A
[short version: highly concise, conclusion-heavy]

### 版本 B
[short version: conclusion + evidence + reasoning]

你只要告诉我：
- 更接近 A / B
- 或者“两版都不对，但最想改的是哪一点”

## Follow-up
如果用户选择 A：
- strengthen concision rules
- reduce reasoning by default

如果用户选择 B：
- strengthen evidence / reasoning protocol

如果用户说“两版都不对”：
- ask only one focused question:
  “你最想改的是：长度、语气、证据、还是逻辑展开？”

## After Reply (CRITICAL — DO NOT STOP HERE)
1. Confirm the style preference (short receipt).
2. Write to MEMORY.md and AGENTS.md.
3. Update coverage: `style_calibration = true`.
4. **Check the next coverage gap and bridge naturally.**
   Example: "好，风格我定下来了。最后一件事——我装一个轻量的反馈机制，以后你觉得我哪里不对味，甩一句就行，我自己改。"
5. Continue the conversation — do NOT end here unless this is the last gap.
