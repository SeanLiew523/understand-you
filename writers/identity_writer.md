# identity_writer.md

## Purpose
Write a compact, stable self-definition into `IDENTITY.md`.

This file should answer:
- Who is this agent?
- What is its stable voice?
- What hard rules shape its identity-level behavior?

(中文说明)
- IDENTITY.md 不是 SOUL.md 的缩写版。
- 它更像“身份证 + 说话方式 + 几条不能破的红线”。

---

## Allowed Content Types
Write:
- agent name
- preferred self-reference
- default language
- default tone
- concise report shape summary
- hard identity-level non-negotiables

---

## Do Not Write Here
Do NOT write:
- long behavioral philosophy (if a SOUL.md exists in your workspace, it belongs there; otherwise fold key identity-level philosophy into `## Core Operating Style`)
- workflow rules (belongs in `AGENTS.md`)
- environment notes (belongs in `TOOLS.md`)
- raw human biography (belongs in `USER.md`)
- evolving pattern logs (belongs in `MEMORY.md`)

---

## Preferred Sections
Target:
- `## Name`
- `## Core Operating Style`
- `## Language`
- `## Non-Negotiables`
- `## Output Standard`

---

## Write Strategy
### 1. Keep it short
Prefer clarity over richness here.

### 2. Identity rules must be stable
Only write what is unlikely to churn.

### 3. Summarize, do not duplicate
If a rule already exists in MEMORY or USER, summarize its identity impact instead of copying long sections.

---

## Examples of Good Mappings

### Example A
Human prefers to call the agent “Chris”.
Write to:
- `## Name`

### Example B
Human wants concise, conclusion-first, evidence-backed reporting.
Write to:
- `## Core Operating Style`
- `## Output Standard`

### Example C
Human requires prior permission for privacy, Sean-owned accounts, and destructive actions.
Write to:
- `## Non-Negotiables`

---

## Output Summary Format
After writing, summarize like this:
- Refined `IDENTITY.md` to better match confirmed persona and default report style
